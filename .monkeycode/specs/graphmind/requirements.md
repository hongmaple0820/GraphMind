# GraphMind 需求规格说明书

> 本地优先 · 图谱驱动 · Agent 智能 · WebDAV 同步

## 1. 概述

### 1.1 产品定位

GraphMind 是一款面向隐私敏感型知识工作者的本地优先 AI 知识库桌面应用。核心愿景：让知识可连接（Graph）、可推理（Agent）、可掌控（Local-first）。

### 1.2 目标用户

- 开发者：需要管理技术文档、代码笔记、项目知识
- 研究员：需要构建文献关系图谱、跨文档推理
- 学生：需要结构化课程笔记与智能问答
- 隐私倡导者：拒绝云端托管，数据完全本地掌控
- 自建云用户：需要 WebDAV/自托管同步方案

### 1.3 差异化价值

1. **离线可用 + 云端可选**：核心功能完全离线运行，WebDAV 同步为可选增强
2. **图谱自动构建与探索**：基于双向链接、标签、语义相似度自动生成知识图谱
3. **本地 LLM 零成本推理**：集成 llama.cpp，支持 GGUF 量化模型本地推理
4. **开放标准 WebDAV 同步**：不绑定特定云服务，兼容 Nextcloud/坚果云等

| 属性 | 描述 |

| **ID** | FR-ED-001 |
| **名称** | Markdown 编辑器 |
| **EARS 模式** | 当用户打开或创建 .md 文件时，系统应提供基于 CodeMirror 6 的 Markdown 编辑器，支持实时预览和语法高亮 |
| **验收标准** | 1. 可打开、编辑、保存 .md 文件 2. 支持常见 Markdown 语法 3. 编辑器响应时间 < 50ms |

## 4. 约束与假设

### 4.1 约束

- C1: 桌面端应用，基于 Electron 框架
- C2: 目标硬件 i5-12600KF + 32GB RAM，无 GPU 要求
- C3: 本地推理使用 CPU，需编译 AVX2/AVX512 优化
- C4: 同步协议限定 WebDAV 标准（RFC 4918）
- C5: 模型格式限定 GGUF（llama.cpp 兼容）

### 4.2 假设

- A1: 用户具备基本的 Markdown 编写能力
- A2: 用户可自行下载或提供 GGUF 模型文件
- A3: WebDAV 服务由用户自行搭建或购买
- A4: 用户设备至少有 10GB 可用磁盘空间用于模型和索引

## 6. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 本地推理卡顿 | 用户体验下降 | 默认 Q4_K_M 量化 + Worker 线程 + 轻量模式(3B) |
| 图谱大库卡顿 | 渲染性能问题 | 虚拟节点渲染 + 按需加载 + 降级静态布局 |
| WebDAV 冲突复杂 | 数据丢失风险 | 默认 local-wins + 云端备份 + Diff 预览 + 撤销同步 |
| 模型兼容性碎片化 | 维护成本高 | 统一 Provider 接口 + 配置驱动注册 |
| 内存泄漏 | 长期运行崩溃 | 定期 Profiling + 模型自动卸载 + Worker 隔离 |
